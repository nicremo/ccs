# Changelog

All notable changes to CCS are documented here.

## 1.4.0

- Expanded the open-source project structure with license, contributing, security, code of conduct, and GitHub templates.
- Reworked the README for Claude Code provider switching, installation, safety, provider sources, and release workflow.
- Added SEO-focused npm keywords for Claude Code, provider switching, and supported model families.
- Improved German UI strings with correct umlauts.

## 1.3.1

- Aligned Claude Code environment defaults with refreshed provider documentation.
- Added `CLAUDE_CODE_AUTO_COMPACT_WINDOW` handling for supported provider presets.
- Added Kimi `ENABLE_TOOL_SEARCH=false`.
- Updated Z.AI default to `glm-5.2[1m]`.

## 1.3.0

- Refreshed the provider catalog for MiniMax, Kimi, Z.AI, DeepSeek, and Qwen.
- Added provider-specific validation headers.
- Added tests for provider defaults, endpoints, and validation headers.
- Updated production and development dependencies to remove known audit findings.

## 1.2.0 and earlier

- Added DeepSeek and Qwen providers.
- Added MiniMax M2.7 and Z.AI GLM model support.
- Renamed the package to CCS.
- Added the first interactive Claude Code provider switching workflow.
